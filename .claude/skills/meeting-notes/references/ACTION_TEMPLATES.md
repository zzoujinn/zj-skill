# 行动项模板和跟踪

本文档提供了创建、跟踪和管理会议行动项的模板和方法。

## SMART 行动项

每个行动项应该是：

- **S**pecific（具体）- 清晰描述需要做什么
- **M**easurable（可衡量）- 结果可以验证
- **A**ssignable（可分配）- 有明确的负责人
- **R**ealistic（现实）- 使用现有资源可以实现
- **T**ime-bound（有时限）- 有明确的截止日期

## 基本行动项模板

```
[动词] + [对象] + [上下文]

示例：
- "为仪表板编写用户认证模块"
- "安排与设计团队的后续会议"
```

## 表格格式

### 标准行动项表格

| # | 行动项 | 负责人 | 截止日期 | 优先级 | 状态 | 备注 |
|---|-------------|-------|----------|----------|--------|-------|
| 1 | [描述] | [姓名] | [日期] | 高/中/低 | 待办/进行中/完成 | [详情] |
| 2 | [描述] | [姓名] | [日期] | 高/中/低 | 待办/进行中/完成 | [详情] |
| 3 | [描述] | [姓名] | [日期] | 高/中/低 | 待办/进行中/完成 | [详情] |

### 简化表格（用于较小的会议）

| 项目 | 负责人 | 截止日期 | 状态 |
|------|-------|----------|--------|
| [描述] | [姓名] | [日期] | 待办 |
| [描述] | [姓名] | [日期] | 待办 |

## 状态值

| 状态 | 描述 |
|--------|-------------|
| 待办 | 未开始 |
| 进行中 | 工作已开始 |
| 阻塞 | 因依赖项无法继续 |
| 待审核 | 准备好审核/批准 |
| 完成 | 已完成 |

## 优先级级别

| 优先级 | 描述 | 示例 |
|----------|-------------|---------|
| 紧急 | 必须现在完成，阻塞进度 | "修复影响用户的生产环境 bug" |
| 高 | 3-5 天内完成 | "完成设计审核" |
| 中 | 1-2 周内完成 | "更新文档" |
| 低 | 尽可能完成 | "重构旧代码" |

## 行动项示例

### 好的行动项
- "在周五之前用新端点更新 API 文档"
- "在今天结束前安排与设计团队的后续会议"
- "在下周二之前审核并批准预算提案"

### 坏的行动项
- "思考问题" - 不具体或不可衡量
- "修复网站" - 太模糊
- "与市场部门交谈" - 没有明确的结果或截止日期

## 与工具集成

### Jira 格式
```

- [DEV-123] 实现用户认证 - 截止: [日期]
```

```
任务: [行动项]
分配给: [姓名]
截止日期: [日期]
项目: [项目名称]
```

```
卡片: [行动项]
```

## 最佳实践

### 创建行动项
1. **会议期间记录** - 不要等待，在分配时立即记录
2. **与负责人确认** - 确保每个人都同意他们的行动项
3. **具体明确** - 清晰的描述避免混淆
4. **设定现实的截止日期** - 考虑到人的工作量

### 跟踪行动项
1. **使用一致的系统** - 所有会议使用相同的格式/工具
2. **定期审核** - 在后续会议上检查状态
3. **庆祝完成** - 当项目完成时给予认可
4. **处理延迟** - 了解项目为何受阻并帮助解除阻碍

### 更新行动项
1. **标记为完成** - 完成时明确标注
2. **更新截止日期** - 如果真的需要，附上解释
3. **必要时重新分配** - 当负责人变更或容量变化时
4. **归档已完成项目** - 保留记录但从主动跟踪中移除

## 行动项审核

定期审查行动项：

```
# 行动项审核 - [月份/季度]

## 完成率
- 总计分配: [X]
- 已完成: [Y] ([Z]%)
- 进行中: [A]
- 已逾期: [B]

## 逾期项目
| 项目 | 负责人 | 原始截止日期 | 当前状态 |
|------|-------|--------------|----------------|
| [项目] | [姓名] | [日期] | [状态] |

## 重复出现的问题
- [识别逾期项目的模式]
- [识别常见的阻塞项]
- [识别需要支持的负责人]

## 需要的改进
- [流程改进]
- [沟通改进]
- [资源分配变更]
```

## 快速参考

### 行动项状态代码
- `[ ]` 或 `待办` - 未开始
- `[~]` 或 `进行中` - 工作进行中
- `[!]` 或 `阻塞` - 无法继续
- `[x]` 或 `完成` - 已完成

### 优先级指示符
- 🔴 紧急
- 🟠 高
- 🟡 中
- 🟢 低

### 行动项常用动词
- 编写、创建、开发、构建
- 审核、批准、验证
- 安排、组织、协调
- 研究、分析、调查
- 更新、修改、修复
- 会面、讨论、咨询
